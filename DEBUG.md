## 待办

### 功能缺口

- **QQ 音乐歌单导入还没做**：分享链接如果是网易云的能正常导入，QQ 音乐链接目前只会提示"开发中"。

### 需要拿主意的问题

- **本机开发环境下 SoundCloud 搜不出来、也放不了**：这台机器现在联网要走公司代理，但项目代码默认直连，所以只有 SoundCloud 会失败（网易云、QQ 不受影响，因为它俩是浏览器直接连 CDN，不需要经过项目自己的代理）。启动服务时加上环境变量 `CORP_PROXY_HOST=proxy.nioint.com` 就能解决，已经验证过。要不要把这个配置固定下来，还没定。
- **SoundCloud 的歌大部分播完不会存到本地缓存**：网易云、QQ 的歌只要播过一次就会自动缓存到本地，下次播放不用再联网。SoundCloud 不一定——大部分歌默认走的是一种叫 HLS 的流媒体格式，现在的缓存逻辑没覆盖这种格式，只有少数走普通 mp3 格式的 SoundCloud 曲目才会被缓存。要解决的话有两个方向：一是把 SoundCloud 也强制改成走普通 mp3（代价是会重新出现一个已经修好的播放失败问题）；二是单独写一套逻辑把 HLS 格式的歌也存下来（工作量明显更大，还没验证可行性）。两个都还没做，需要再合计一下选哪个。

### 已经讨论过、暂时不做的

- QQ 音乐搜索结果想要"搜出来就有封面"（现在是点开播放才会补全封面），技术上可行但会让每次搜索多打很多次请求，等以后有空再评估要不要做。

## 当前状态

播放器主要功能都能正常用：三个音源聚合搜索（网易云、QQ 音乐、SoundCloud）、真实播放（进度/音量/播放模式/播完自动切歌）、播放队列（拖拽排序、移除）、歌单（新建、删除、加入/移出、喜欢，都会存在本地，刷新页面不丢）、歌单可以导出成一段字符串（复制到剪贴板）、导入支持网易云分享链接或这个导出字符串（覆盖式：同名歌单直接替换，留空则整体替换"喜欢的音乐"）、歌词滚动高亮、明暗主题切换、歌曲和封面在浏览器本地自动缓存。已经部署上线：`https://listening-5bnv.onrender.com/`。

酷我、JOOX 两个音源之前用过，已经下线不再维护，固定只用网易云、QQ 音乐、SoundCloud 三个源。

## 变更记录

按时间倒序，只记做了什么，不记详细的排查过程（想看细节可以翻 git log / commit）。

**2026-07-10**
- 加了"导出歌单"：歌单详情页跟"播放全部"同一排新增按钮，点一下把这个歌单编码成一段字符串（`LSNPL1:` 开头，内容是精简 JSON 的 base64）复制到剪贴板，按钮文字临时变成"已导出到剪贴板"。"导入歌单"对话框相应支持粘贴这段字符串（原来只认网易云分享链接）。
- 顺带把导入行为从"新建/叠加"改成"覆盖"：填了歌单名且跟已有非内置歌单同名，会直接替换那个歌单的内容而不是再建一个重名的；名字留空则把"喜欢的音乐"整个替换成这次导入的内容（不是合并，之前喜欢但不在这次导入里的歌会被摘掉——这是跟用户确认过的有意行为，对话框小字里也写清楚了）。用 Playwright 在真实浏览器里跑通了导出（含真实剪贴板 API 读写）和两种覆盖场景的验证。
- 修了"手机浏览器切到后台、当前歌曲播完不会自动切下一首"：根因是代码里从没注册过 Media Session（`navigator.mediaSession`），移动端浏览器没有"这个标签页在持续播放媒体"的系统级信号，切后台后可能限流/拦截 `ended` 事件里异步发起的下一首播放请求。加上了播放控制的 Media Session 注册（连带获得锁屏/通知栏的媒体控件）、每次换曲目同步歌曲元数据，顺手去掉了切歌路径上一个多余的 `setTimeout(0)` 延迟。这台环境没法模拟手机切后台的限流场景，只做了本地静态检查和桌面浏览器跑起来的基本验证，真实效果还需要在手机上背景播放实测确认。
- 排查 SoundCloud 搜索/播放失败和本地缓存缺口两个问题，定位到原因，记进上面的"待办"，暂未动代码。

**2026-07-09**
- 部署上线到 Render，加了防止服务休眠的保活机制。
- 把原来两个独立进程（静态页面服务 + 代理服务）合并成一个，代理是否走公司内网隧道改成按环境变量开关（默认直连，公网部署不需要额外配置）。

- 新用户首次打开不再显示假数据（假歌单、假"正在播放"）。
- 支持从网易云分享链接导入歌单（QQ 音乐链接还不支持）。
- 播放队列和歌单曲库的数据结构拆开了，修复"播放全部"会误改动歌单本身的问题。

**几个容易忘记的背景知识**

- SoundCloud 播放优先走 HLS 格式，是为了绕开另一种格式（progressive mp3）在部分曲目上会被 CDN 拒绝访问的问题。
- 本机如果所在网络需要走公司代理才能联网，启动服务要加 `CORP_PROXY_HOST=proxy.nioint.com` 环境变量，SoundCloud 相关功能才能用；线上部署（Render）不需要这个配置。
- 列表/网格视图切换、酷我音源、JOOX 音源，都是评估过后用户决定不做/下线的，不是遗漏。
